# 立体分析法深度研究

> **方法论溯源**
> 立体分析法融合了系统动力学中的因果回路图、商业战略中的情景规划、以及反事实思维的核心思想。X轴追踪因果链，Y轴还原多因素并行，Z轴推断反事实路径——三轴交汇，才能产生真正立体、有纵深的洞察。

你正在执行一次立体分析法深度研究。最终产出一份**排版精美的PDF研究报告**。

## 核心框架

三个维度回答三个不同的问题：

**X轴 — 时间线因果链**：这件事是怎么一步步走到今天的？
- 沿着时间轴还原因果链
- 每个节点：发生了什么 → 为什么重要 → 导向下一个节点的机制是什么

**Y轴 — 多因素并行展开**：在每个关键节点，有哪些力量同时在作用？
- 竞品、资本、技术、人物、用户、市场…在同一时间截面并行存在
- 不是简单罗列，而是呈现多股力量如何交织、博弈、共振

**Z轴 — 反事实路径推断**：在真正的岔路口，是什么锁定了路径？
- 识别每个关键节点存在的真实替代选择
- 推断如果走了另一条路，会通向什么结果
- 分析什么机制（网络效应、沉没成本、生态锁定…）让这条路径难以逆转

## 第一步：联网信息收集

这个方法论的质量完全取决于信息的丰富度和准确性。**必须联网搜索**，不能仅靠已有知识。研究报告的价值在于深度和完整度，所以信息收集阶段宁可多搜，不要因为信息不够导致后面的分析浮于表面。

### 并行搜索策略

使用子Agent并行搜索来提高效率。建议的分工：

- **子Agent 1 — X轴信息**：研究对象的起源、发展历程、关键事件、版本迭代、融资、战略转向、危机
- **子Agent 2 — Y轴信息**：在各个关键节点上，竞品、资本、技术、市场、人物等力量各自的动态
- **子Agent 3 — Z轴补充**：创始人深度背景、行业环境变化、用户社区讨论（GitHub Issues、Reddit、Twitter/X、知乎等）、以及可用于反事实推断的对照案例

**子Agent联网工具使用指南**（直接写入每个子Agent的prompt中）：

每个子Agent的prompt中必须包含以下联网指引：

> 你需要联网获取信息。使用以下工具：
> - **WebSearch**：用于搜索发现信息来源，获取摘要和关键词结果
> - **WebFetch**：当已知具体URL时，用于从页面定向提取内容
> - 如果用户环境中安装了 web-access skill（检查路径 `/mnt/.claude/skills/web-access/SKILL.md` 是否存在），优先加载它并遵循其指引，它提供更强的浏览器CDP能力
> - 搜索策略：先用WebSearch发现信息来源和线索，找到具体URL后用WebFetch深入提取
> - 多次搜索、多个关键词组合，不要只搜一次就放弃
> - 一手来源优于二手来源：官方博客 > 权威媒体原创报道 > 转载/聚合
> - **学术类研究对象必查arxiv**：如果研究对象涉及学术概念、算法、AI模型、技术范式等，必须通过arxiv API获取相关论文。调用方式：`curl -s "https://export.arxiv.org/api/query?search_query=all:关键词1+AND+all:关键词2&max_results=10"`，或用WebFetch访问同一URL。返回XML格式，包含标题、作者、摘要、发布日期、PDF链接。可按需调整关键词组合和结果数量。找到关键论文后，用WebFetch读取论文页面（`https://arxiv.org/abs/论文ID`）获取更多细节。

prompt要描述目标（"获取""调研""了解"），不要用暗示具体手段的动词（"搜索""爬取"），让子Agent自主判断最佳获取方式。

### 信息来源优先级

一手来源优于二手来源，多个媒体引用同一个错误会造成循环印证假象：

| 信息类型 | 一手来源 |
|---------|---------|
| 产品更新/技术决策 | 官方博客、GitHub Release Notes、创始人推文 |
| 融资/商业数据 | 公司官方公告、SEC/工商文件 |
| 用户口碑 | GitHub Issues、Reddit讨论、Twitter/X、知乎帖子 |
| 行业分析 | 权威媒体原创报道（非转载） |
| 学术/技术原理 | arXiv论文（`export.arxiv.org/api/query`）、Google Scholar、学术会议论文集 |
| 反事实参照 | 同类竞品的不同路径、行业历史案例、学术研究中可供对照的证据 |

### 信息充分性自检

搜索完成后检查：
- X轴：能讲出一条完整的因果链吗？每个节点的"因为→所以"清楚吗？
- Y轴：在主要时间节点上，力量并行展开的信息够吗？有没有某条线明显缺失？
- Z轴：有没有识别出真正的岔路口？反事实推断有没有参照系？
- 来源：关键事实有可靠来源支撑吗？有没有只靠单一来源就下判断的？

信息不够就再补搜。不要凑合。

## 第二步：X轴分析 — 时间线因果链

沿时间轴还原研究对象从诞生到现在的发展全貌。不是流水账，而是因果链：每个节点发生了什么、为什么导向下一个节点。

### 内容要求

**起源追溯**：它诞生的背景是什么？基于什么技术/理念/需求而来？创始团队或核心推动者是谁？这些人之前做过什么，为什么是他们？当时的行业环境是什么样的？

**因果链构建**：从起源到当下，每个关键节点不是孤立的事件罗列，而是要回答：
- 这个节点发生的直接原因是什么？
- 它导向下一个节点的机制是什么？（技术路线选择、市场反馈、竞争压力、资源积累…）
- 推动力从哪来？阻力从哪来？

**关键节点识别**：不是所有事件都等重要。识别真正改变走向的节点——可能是：
- 融资节点（拿到钱或失去钱）
- 产品转折点（发布、转型、放弃）
- 创始人/核心人物变动
- 外部冲击（政策、竞品动作、市场变化）
- 战略转向

**阶段划分**：把整个历程自然分为几个阶段，每个阶段有核心驱动力和核心矛盾。

### 篇幅

4000-12000字。核心原则是因果链清晰、每个关键节点的"因→果"关系明确。不要为了字数堆砌细节，要让读者感受到事件之间的必然性。

---

## 第三步：Y轴分析 — 多因素并行展开

以X轴为基础，在每个关键时间节点上，还原当时有哪些力量同时在作用、它们之间如何交织。

### 力量类型

根据研究对象类型，力量类型可能包括：

| 类型 | 说明 |
|------|------|
| 竞品 | 同期竞争对手在做什么 |
| 资本 | 融资节奏、投资方背景、资金压力 |
| 技术 | 技术路线、基础设施成熟度、专利布局 |
| 市场 | 需求变化、用户认知、渠道变化 |
| 政策/监管 | 行业法规、政府的态度 |
| 人物 | 关键人物变动、团队稳定性 |
| 舆论/口碑 | 社区反应、媒体叙事、用户评价 |

### 分析要求

**并行呈现**：在每个关键节点，不是把这些力量一一罗列，而是呈现它们之间的**交织、博弈、共振**。

例如：2007年iPhone发布时——
- 诺基亚：不屑一顾（竞品反应）
- Google：悄悄买下Android、调整战略（竞品的暗中布局）
- AT&T：独家合作锁定了早期用户（资本+渠道）
- 触控屏技术：已经成熟（技术条件）
- App Store：无人理解的全新模式（市场认知）

这些不是孤立罗列，而是呈现它们如何**在同一时刻共同塑造了智能手机行业的起点**。

**力量互动**：每种力量不是静态的，它们之间存在因果关系：
- A力量的变化如何影响B力量？
- 谁是主动的施动者，谁是被动的响应者？
- 哪些力量是背景板，哪些是主动塑造格局的？

### 篇幅

3000-8000字。根据关键节点数量和力量复杂度调整。

## 第四步：Z轴分析 — 反事实路径推断

这是立体分析的灵魂。在X轴的因果链上，识别真正的岔路口，推断反事实路径。

### 什么是真正的岔路口

不是所有"选择"都值得分析。只有满足以下条件的节点才值得进入Z轴：

1. **真实的选择**：当时确实存在不止一条可行路径
2. **可推断的后果**：有参照系可以推断走了另一条路会怎样
3. **路径锁定机制**：是什么让最终走向的这条路径难以逆转？

### 分析框架

对每个关键岔路口：

**选项A（实际发生）**：走了X轴路径，结果是什么？
**选项B/C（反事实）**：走了替代路径，结果可能是什么？
**为什么锁定**：什么机制（网络效应、沉没成本、生态锁定、法律壁垒…）让这条路越走越窄、无法回头？

### 常见锁定机制

- **网络效应**：越多人用越有价值，形成正反馈
- **沉没成本**：已经投入的资源让人不愿退出
- **生态锁定**：围绕它建立的上下游、配套、用户习惯难以迁移
- **技术路线依赖**：选了某条技术路线后，基础设施、人才、工具链都围绕它建立
- **制度/法律壁垒**：监管牌照、专利墙、政策保护
- **认知壁垒**：用户心智被占领，后来者需要说服的不只是功能，还有认知

### 反事实推断的约束

**必须基于事实，不能空想。** 可用的推断依据：
- 同类竞品走了不同路径的结果（横向参照）
- 历史先例（纵向参照）
- 学术研究或专家判断
- 可信的逻辑推演

如果缺乏依据，明确标注"此处为推测"，不要把推测当结论。

### 篇幅

2000-5000字。不是每个节点都需要Z轴，只写真正的岔路口。宁可少写几个，写深写准。

## 第五步：三轴交汇洞察

这是整篇报告的精华段。把X、Y、Z三条轴的信息编织在一起，产出超越单轴的综合性判断。

### 核心问题

1. **因果链如何塑造了今天**：X轴上的哪些因果关系，决定了研究对象今天的位置？
2. **多力量格局如何影响了因果**：Y轴上的哪些力量博弈，加速或延缓了因果链？
3. **岔路口的教训是什么**：Z轴的分析，给我们什么关于"路径选择"的洞察？
4. **未来推演**：基于三轴分析，最可能的剧本、最危险的剧本、最乐观的剧本各是什么？

### 判断 vs 描述

交汇洞察不是描述性内容的缩写版，而是**新的判断**。每个洞察应该能用一两句话说清楚，而不是"综上所述…"

### 篇幅

1500-3000字。精华不靠长度，靠洞察的锐度。

## 写作风格

这不是一份冷冰冰的咨询报告，而是一篇让人能从头读到尾的深度研究。写作风格需要在「研究报告的严谨」和「有温度的可读性」之间找到平衡点。

### 核心原则

**节奏感**：句子时长时短，段落之间跳跃自然。不要每段都一样长。好的节奏像波动，围绕主线偏出去一点，再用一句「扣主线句」拉回来。

**叙事驱动，不是罗列驱动**：X轴部分要有故事弧线，有起承转合。不要写成"2023年1月发布了A，2023年3月发布了B"的流水账。

**知识是「聊着聊着顺手掏出来」的**：在讲述过程中自然地带出背景知识，不要「下面我来给大家科普一下」。

**敢下判断**：鼓励给出观点和洞察，但每个观点必须有事实支撑。先摆事实，再给判断。是推测的明确标注。表达判断时用「我觉得」「我的判断是」这种承认主观性的姿态。

**层层剥开的修辞**：不直接讲结论，用"现象→表面解释→更深的追问→核心洞察"的方式展开。

**回环呼应**：开头埋的细节和钩子，在结尾callback回来。前后因果的闭合感，是让报告从「信息流」变成「作品」的关键。

### 绝对禁区

以下AI味标记无论什么文体都要避免：
- 套话："首先...其次...最后"、"综上所述"、"值得注意的是"、"不难发现"
- 空洞形容词："赋能"、"抓手"、"打造闭环"
- 教科书开头："在当今AI快速发展的时代"、"随着技术的不断进步"
- 高频踩雷词："说白了"、"意味着什么？"、"这意味着"、"本质上"、"换句话说"、"不可否认"
- 空泛工具名：不说"AI工具"、"某个模型"，要说具体名字
- 编造场景：如果某个信息搜不到，诚实标注「该信息暂缺」，绝不编造

### 用人话写

避免咨询公司式的套话和空洞概括。用具体的细节和例子代替概括性陈述。比如不要写「该公司在这一阶段实现了快速增长」，而要写「从2024年中期的1000万美元ARR到2025年底的10亿美元，增长曲线几乎是垂直的」。

## 第六步：生成PDF报告

报告写完后，使用本Skill自带的 `scripts/md_to_pdf.py` 脚本将Markdown转为排版精美的PDF。

### 转换流程

1. **先完成Markdown稿件**：将完整报告写为标准Markdown格式，保存为 `[研究对象]_立体分析报告.md`
2. **安装依赖**（如未安装）：`pip install weasyprint markdown --break-system-packages`
3. **运行转换脚本**：
   ```bash
   python [skill目录]/scripts/md_to_pdf.py input.md output.pdf --title "研究对象名称" --author "jf-analysis"
   ```
4. 脚本会自动生成中间HTML文件（便于调试）和最终PDF

### 脚本内置的排版规范

`md_to_pdf.py` 已内置完整的CSS排版方案，无需手动调整：

- **页面**：A4，页边距上25mm/左右20mm/下20mm
- **封面页**：自动生成，包含标题（28pt深蓝色）、副标题「立体分析法深度研究报告」、作者信息、装饰分隔线
- **配色**：H1标题=#1a5276深蓝、H2=#1e8449绿色、H3=#2e86c1浅蓝、H4=#5b2c6f紫色，正文=#2c3e50深灰
- **字体**：CSS fallback链 `"Droid Sans Fallback", Helvetica, Arial, sans-serif`，自动处理中英文混排
- **正文**：10.5pt，行距1.75，两端对齐，孤行/寡行控制
- **引用块**：左侧3pt深蓝竖线 + 浅灰背景
- **表格**：全宽、深蓝表头白字、斑马纹行
- **页眉**：「报告标题 | 立体分析法深度研究报告」（首页不显示）
- **页脚**：「第 X 页」（首页不显示）
- Markdown的第一个H1会被自动提取为封面标题，正文中不会重复出现

### Markdown写作注意事项

为了让脚本正确解析并生成最佳PDF效果：

- 第一行用 `# 标题` 作为报告标题（会自动用于封面）
- 紧接标题后可用 `> 研究时间：... | 所属领域：... | 研究对象类型：...` 格式写元信息行，会被提取到封面
- 用 `##` 作为主要章节标题（X轴分析、Y轴分析、Z轴分析、交汇洞察等）
- 用 `###` 和 `####` 作为子章节
- 表格使用标准Markdown表格语法
- 引用使用 `>` 语法
- 加粗使用 `**文本**`

### 末尾内容

在Markdown稿件末尾加上：
- **信息来源**：所有引用的来源清单，标注URL和访问时间
- **方法论说明**：简要说明立体分析法的框架来源（1-2句话即可）

## 不同研究对象类型的适配

核心框架不变（X因→Y力→Z反事实），但侧重点不同：

**研究产品时**：X轴重点关注版本迭代、用户增长曲线、关键产品决策；Y轴重点关注竞品动态、技术演进、用户口碑的并行力量；Z轴重点关注技术路线选择、平台生态锁定的反事实可能。

**研究公司时**：X轴重点关注创始团队、融资历程、战略转向；Y轴重点关注资本结构、行业格局、关键人物的并行力量；Z轴重点关注战略转折点的替代路径和组织路径依赖。

**研究概念时**（技术范式、商业模式、文化现象）：X轴重点关注概念的起源和演变；Y轴重点关注不同阵营、相关技术的并行力量；Z轴重点关注概念竞争背后的路径锁定。

**研究人物时**：X轴重点关注个人经历、关键决策、成长曲线；Y轴重点关注同期人物、环境变化的并行力量；Z轴重点关注人生岔路口的替代可能。

## 篇幅总览

| 部分 | 字数范围 | 说明 |

| X轴分析 | 4,000 - 12,000字 | 因果链清晰，每个节点有因有果 |

| **全文总计** | **10,000 - 28,000字** | 深度和完整度是价值所在 |
