# 卓越运营 PPT 生成

使用 `assets/template.pptx` 作为唯一视觉模板，为用户主题生成规范企业风格 PPT。默认同时输出 `.pptx` 和 `.html`：PPTX 用于正式汇报和二次编辑，HTML 用于快速预览、校对标题链和页面内容。

## 默认输出

- 默认生成 12 页：1 页开头页、10 页核心内容页、1 页结束页。
- 用户指定页数时，以用户指定为准。
- 除非用户明确只要一种格式，否则同时生成 `.pptx` 和 `.html`。
- PPTX 和 HTML 必须来自同一份 deck JSON，避免内容不一致。

## 不可违反规则

- 观点在标题上，标题使用简短语句表达。
- 每页传递观点三点以内。
- 中文字体使用微软雅黑，英文字体使用 Times New Roman。
- 每页字号三种以内。
- 决不允许出现动画和页面转场。
- 页面必须符合模板视觉规范。
- 关键内容标红加粗，但只强调关键词、核心数字、风险点或行动重点。
- 慎用感叹号，默认不使用感叹号。

## 模板视觉规则

### 字体与字号

- 中文使用微软雅黑，英文使用 Times New Roman。
- 每页字号不超过三种。
- 页面主标题使用 20-24 号字。
- 卡片一级标题使用 11 号字。
- 卡片正文使用 10 号字。
- 卡片二级标题使用 10 号字。
- 二级正文使用 9 号字。
- 标签、注释、图例、弱化信息常用 8 号字。
- 非重点辅助注释最小可使用 6 号字。
- 内容较多时，大段文字不建议使用 11 号或更大字号。

### 文本颜色与强调

- 常规正文使用黑色。
- 强调内容使用红色。
- 非重点内容、注释、辅助标签使用灰色。
- 红色标题栏中的标题使用白色反白。
- 页面标题、卡片一级标题可以加粗。
- 关键数字、关键词、风险点、行动重点可以标红加粗。
- 小标题和正文不建议频繁加粗。
- 禁止整段正文加粗或整段正文标红。
- 如果一页中强调内容过多，必须减少到 1-3 个最重要的信息点。

### 段落样式

- 默认段落行距为 1.3 倍。
- 特殊情况下可调整为 1.5 倍或 2.0 倍。
- 正文段落使用两端对齐。
- 段前间距一般为 0。
- 段后间距一般为 12，可根据内容量调整为 3、6、12、18。
- 项目符号和编号一般使用默认样式。
- 避免大段文字堆叠，优先拆成短句、分组或卡片。

### 卡片、线条与组件

- 卡片内文本需要保留稳定边距。
- 不使用项目符号或编号时，左右边距保持一致。
- 常用左右边距为 0.4、0.6、0.8 厘米。
- 使用项目符号或编号时，左边距通常比右边距少 0.2 厘米。
- 箭头线、连接线、分割线、卡片轮廓线优先使用 0.5 磅。
- 线条颜色优先使用浅灰、深灰、黑色或红色。
- 卡片轮廓以浅灰框、深灰框、红色框为主。
- 卡片填充以白色、浅灰、红色系为主。
- 箭头大小一般选择最小。
- 优先复用模板中的箭头、连接线、分割线和卡片样式。

### 配色与图表

- 默认使用红色系和灰色系。
- 主红用于页面标题、重点强调、红色标题栏和核心视觉元素。
- 灰色用于边框、分割线、弱化信息、辅助图形和非激活状态。
- 蓝色、黄色、绿色只在需要区分类别时使用，例如架构图、多分类图表。
- 不建议大量使用红色，避免页面混乱。
- 不建议滥用渐变色。
- 柱状图优先使用红色系或灰色系。
- 对比类图表使用红色突出关键系列，其余系列用灰色弱化。
- 饼图、环图优先使用同一色系的深浅变化。
- 图表颜色必须服务于结论，不为了装饰而多色。

## 工作流

1. 理解用户主题、用途、受众、资料来源、页数和输出语言。
2. 先生成标题链，再生成每页正文。标题链必须能串成一条完整故事线。
3. 根据 `references/template-map.md` 为每页选择模板页和页面角色。
4. 将页面规划保存为统一 deck JSON。
5. 使用 `scripts/build_ppt.py` 从 `assets/template.pptx` 生成 `.pptx`。
6. 使用 `scripts/build_html.py` 从同一份 deck JSON 生成 `.html`。
7. 使用 `scripts/export_preview.py` 导出 PPT 预览图，检查错位、溢出、空白和信息过载。
8. 返回 `.pptx` 和 `.html` 文件路径，说明两者用途。

## Deck JSON

脚本使用同一份 JSON 作为输入：

```json
{
  "title": "AI客服运营效率提升方案",
  "subtitle": "管理层汇报",
  "slides": [
    {
      "template_slide": 13,
      "role": "背景判断",
      "title": "客服运营效率提升进入流程重构阶段",
      "points": [
        {
          "heading": "当前瓶颈",
          "body": ["人工处理占比高", "跨系统流转耗时长"]
        },
        {
          "heading": "优化方向",
          "body": ["智能分流前置", "标准处置流程沉淀"]
        }
      ]
    }
  ]
}
```

## 质量检查

- 每页标题是否表达明确观点，而不是栏目名。
- 每页核心观点是否不超过三点。
- 正文是否直接支撑标题。
- 是否保留模板原有字体、颜色、卡片、线条和布局。
- 是否有文字溢出、元素遮挡、页面空白或明显错位。
- 是否同时生成 PPTX 和 HTML。
