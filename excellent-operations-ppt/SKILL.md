# 卓越运营 PPT 生成

使用 `assets/template.pptx` 作为页面版式库和组件素材库，为用户主题生成规范企业风格 PPT。默认同时输出 `.pptx` 和 `.html`：PPTX 用于正式汇报和二次编辑，HTML 用于快速预览、校对标题链和页面内容。

## 核心定位

- 不把 `template.pptx` 当作固定母版套用。
- 把 `template.pptx` 当作可挑选的页面案例库、版式库和组件素材库。
- 先判断每页要表达什么，再选择整页版式或局部组件。
- 优先复用模板已有视觉语言，不自行创造新的页面风格。
- 交付目标是可汇报、可编辑、可预览的高质量 PPT，而不是脚本成功生成文件。

## 输出规则

- 除非用户明确只要一种格式，否则同时生成 `.pptx` 和 `.html`。
- PPTX 和 HTML 必须来自同一份页面蓝图或 deck JSON，避免内容不一致。
- HTML 用于快速预览标题链、页面结构和内容密度，不替代 PPTX。
- PPTX 必须能在 PowerPoint 中正常打开，并导出预览图检查。

## 页数规则

- 不固定默认 12 页。
- 根据用户材料复杂度、内容密度、表达目标和可用素材决定页数。
- 用户未指定页数时，先规划 8-16 页。
- 内容少时控制在 6-8 页。
- 标准汇报控制在 10-14 页。
- 方法型、培训型、素材丰富型内容可扩展到 15-24 页。
- 不为了凑页数拆水页，也不为了压缩页数牺牲可读性。
- 每页必须承担明确表达任务；没有表达任务的页面必须删除。

## 不可违反规则

- 观点在标题上，标题使用简短语句表达。
- 每页传递观点三点以内。
- 中文字体使用微软雅黑，英文字体使用 Times New Roman。
- 每页字号三种以内。
- 决不允许出现动画和页面转场。
- 页面必须符合模板视觉规范。
- 关键内容标红加粗，但只强调关键词、核心数字、风险点或行动重点。
- 慎用感叹号，默认不使用感叹号。

## 模板视觉规则

### 字体与字号

- 中文使用微软雅黑，英文使用 Times New Roman。
- 每页字号不超过三种。
- 页面主标题使用 20-24 号字。
- 卡片一级标题使用 11 号字。
- 卡片正文使用 10 号字。
- 卡片二级标题使用 10 号字。
- 二级正文使用 9 号字。
- 标签、注释、图例、弱化信息常用 8 号字。
- 非重点辅助注释最小可使用 6 号字。
- 内容较多时，大段文字不建议使用 11 号或更大字号。

### 文本颜色与强调

- 常规正文使用黑色。
- 强调内容使用红色。
- 非重点内容、注释、辅助标签使用灰色。
- 红色标题栏中的标题使用白色反白。
- 页面标题、卡片一级标题可以加粗。
- 关键数字、关键词、风险点、行动重点可以标红加粗。
- 小标题和正文不建议频繁加粗。
- 禁止整段正文加粗或整段正文标红。
- 如果一页中强调内容过多，必须减少到 1-3 个最重要的信息点。

### 段落样式

- 默认段落行距为 1.3 倍。
- 特殊情况下可调整为 1.5 倍或 2.0 倍。
- 正文段落使用两端对齐。
- 段前间距一般为 0。
- 段后间距一般为 12，可根据内容量调整为 3、6、12、18。
- 项目符号和编号一般使用默认样式。
- 避免大段文字堆叠，优先拆成短句、分组或卡片。

### 卡片、线条与组件

- 卡片内文本需要保留稳定边距。
- 不使用项目符号或编号时，左右边距保持一致。
- 常用左右边距为 0.4、0.6、0.8 厘米。
- 使用项目符号或编号时，左边距通常比右边距少 0.2 厘米。
- 箭头线、连接线、分割线、卡片轮廓线优先使用 0.5 磅。
- 线条颜色优先使用浅灰、深灰、黑色或红色。
- 卡片轮廓以浅灰框、深灰框、红色框为主。
- 卡片填充以白色、浅灰、红色系为主。
- 箭头大小一般选择最小。
- 优先复用模板中的箭头、连接线、分割线和卡片样式。

### 配色与图表

- 默认使用红色系和灰色系。
- 主红用于页面标题、重点强调、红色标题栏和核心视觉元素。
- 灰色用于边框、分割线、弱化信息、辅助图形和非激活状态。
- 蓝色、黄色、绿色只在需要区分类别时使用，例如架构图、多分类图表。
- 不建议大量使用红色，避免页面混乱。
- 不建议滥用渐变色。
- 柱状图优先使用红色系或灰色系。
- 对比类图表使用红色突出关键系列，其余系列用灰色弱化。
- 饼图、环图优先使用同一色系的深浅变化。
- 图表颜色必须服务于结论，不为了装饰而多色。

## 素材选择流程

1. 先判断每页表达任务：对比、分类、流程、时间轴、能力架构、系统关系、证据矩阵、方法总览、总结收束。
2. 从 `references/template-map.md` 中选择最匹配的整页版式。
3. 如果没有完全匹配的整页版式，选择相近版式并补充组件素材。
4. 避免连续使用同类版式。
5. 同一套 PPT 中，同一模板页原则上不重复使用；必须重复时，需要改变信息结构和视觉重心。
6. 优先使用模板已有组件，不自行创造新视觉语言。
7. 生成前输出页面蓝图，说明每页使用哪些模板页或组件。

## 页面蓝图规则

生成 PPTX 前必须先形成页面蓝图。每页蓝图至少包含：

- 页面角色
- 观点标题
- 表达类型
- 主要素材来源
- 辅助组件来源
- 信息密度
- 页面结构说明
- 正文内容块

表达类型包括：

- 两类对比
- 三类分类
- 四类并列
- 流程路径
- 时间演进
- 能力架构
- 系统关系
- 证据矩阵
- 方法总览
- 总结收束

页面蓝图示例：

```json
{
  "page": 1,
  "role": "方法定位",
  "title": "立体分析法把复杂对象研究转化为三轴判断系统",
  "expression_type": "方法总览",
  "primary_asset": {
    "type": "full_slide",
    "source_slide": 23
  },
  "supporting_assets": [
    {
      "type": "component",
      "source_slide": 5,
      "asset_id": "red_number_dot"
    }
  ],
  "density": "high",
  "structure": "左侧解释方法定位，右侧用 X/Y/Z 分层结构呈现研究框架",
  "content_blocks": []
}
```

## 工作流

1. 理解用户主题、用途、受众、资料来源、内容规模和输出语言。
2. 先判断内容规模，再决定页数范围；不要默认固定 12 页。
3. 先生成标题链，再生成每页正文。标题链必须能串成一条完整故事线。
4. 为每页判断表达类型，并基于 `references/template-map.md` 选择整页版式或局部组件。
5. 生成页面蓝图，检查页数、版式变化、内容密度和素材调度是否合理。
6. 将页面蓝图转换为统一 deck JSON 或 PPTX 生成输入。
7. 生成 `.pptx` 和 `.html`。
8. 使用 `scripts/export_preview.py` 导出 PPT 预览图。
9. 根据不可交付条件检查预览图；不合格时必须修正后再交付。
10. 返回 `.pptx` 和 `.html` 文件路径，说明两者用途。

## 不可交付条件

出现以下任一情况时，不得交付，必须重做或修正：

- 页面明显空洞，只放 1-2 个短 bullet 且没有结构表达。
- 多页连续使用同一种低密度版式。
- 整份 PPT 没有使用流程图、时间轴、架构图、矩阵或多栏结构中的至少两类。
- 没有使用 `template.pptx` 中的页面版式或组件素材。
- 原模板占位文字未清理干净。
- 文字压住线条、图表、卡片或其他图形。
- 标题不是观点句。
- 页面内容不能支撑标题。
- HTML 和 PPTX 内容不一致。
- 导出的预览图存在文字溢出、遮挡、裁切或明显错位。

## 质量检查

- 标题是否表达明确观点，而不是栏目名。
- 标题链是否形成完整故事线。
- 页数是否由内容复杂度决定，而不是机械固定。
- 每页是否有明确表达任务。
- 是否避免连续重复同类版式。
- 是否充分调用模板中的整页版式和组件素材。
- 每页核心观点是否不超过三点。
- 正文是否直接支撑标题。
- 是否保留模板原有字体、颜色、卡片、线条和布局语言。
- 是否有文字溢出、元素遮挡、页面空白或明显错位。
- 是否同时生成 PPTX 和 HTML。
