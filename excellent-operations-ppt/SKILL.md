# 卓越运营 PPT 设计系统

本 Skill 用 `DESIGN.md` 指导企业汇报类页面的设计与生成。默认交付 `HTML` 和 `PPTX` 两种文件：HTML 用于快速预览和校对，PPTX 用于正式汇报和后续编辑。

本 Skill 不捆绑 PPTX 模板，不做固定母版填充。它的核心任务是读取设计系统，基于用户主题和素材选择合适的版式、组件、色彩、排版和信息密度，生成一份可交付的演示文稿。

## 设计源选择

1. 如果用户指定某个 `DESIGN.md`，优先读取该文件。
2. 如果用户没有指定设计源，读取本 Skill 目录下的 `DESIGN.md`。
3. 如果上下文中存在多个 `DESIGN.md`，先判断主设计系统；除非用户明确要求混合，否则只使用一个主设计系统控制视觉语言。
4. 用户额外提供 PPTX、PDF、图片、网页或文字资料时，将其作为内容和素材参考，不覆盖主 `DESIGN.md` 的质量规则。
5. 主 `DESIGN.md` 是页面版式、组件、配色、字体、段落、图表和质量判断的主要依据。

## 默认交付物

- `HTML`：浏览器预览版本，用于检查标题链、内容结构、信息密度、视觉方向和页面完整度。
- `PPTX`：正式可编辑版本，用于汇报、二次修改和分发。
- 除非用户明确只要一种格式，否则必须同时交付 HTML 和 PPTX。
- HTML 和 PPTX 必须来自同一份页面蓝图或内容模型，不能分别构思两套页面。
- 如果当前环境无法生成 PPTX，必须说明原因，并交付 HTML 与可转换的页面蓝图。

## 工作流

1. 读取主 `DESIGN.md`，提取可执行规则。
2. 理解用户主题、受众、汇报目的、素材规模和时间要求。
3. 先做页数预算，再做页面蓝图；默认优先压缩成 8-12 页，只有内容复杂、素材充分且用户需要完整汇报时才扩展到 13-16 页。
4. 先生成标题链和页面蓝图，明确每页观点、信息层级、推荐版式、素材需求和合并/删减依据。
5. 每页只承担一个主要表达任务，避免把多个复杂观点塞进同一页。
6. 优先生成 HTML 预览或页面结构预览，用于校对内容密度、视觉方向和页面垂直平衡。
7. 基于同一页面蓝图生成 PPTX，逐页对齐 HTML 的标题、正文、图表和重点标注。
8. 最后检查不可交付条件，再给出 HTML 与 PPTX 文件路径。

## 内容规则

- 观点写在标题上，标题必须表达判断、结论或行动方向。
- 正文使用简短但有信息量的语句表达，避免长段落，也避免只有空泛短句。每个主要内容块应至少包含“事实/判断/影响”中的两类信息。
- 每页传递的核心观点不超过三点。
- 关键内容使用红色加粗，但不能满页泛红。
- 慎用感叹号，企业汇报默认不用感叹号。
- 不为了凑页数制造空洞页面，也不因为素材多就机械堆叠。
- 页数由内容决定，但必须先压缩再扩展。标准分析报告优先控制在 8-12 页；超过 12 页必须说明新增页面承担的独立表达任务。
- 长报告转 PPT 时应合并附录、证据和背景，只保留对汇报判断有贡献的内容。
- 内容偏少时优先补充具体数字、来源事实、业务影响、反向约束或观察信号；不要用品牌署名、装饰短语或大留白填充页面。

## 视觉规则

- 中文字体使用微软雅黑。
- 英文字体使用 Times New Roman。
- 每页字号控制在三种以内。
- 禁止使用动画、切换特效和依赖播放顺序才能理解的内容。
- 遵循主 `DESIGN.md` 的色彩、卡片、图表、表格、图标和版式规则。
- 页面必须有明确层级：标题区、主体区、辅助信息区不能混在一起。
- 主体内容必须在标题下方、页脚来源上方的可用画布中垂直居中或视觉居中，不能整体贴近页面上沿、下沿或某一侧。
- 同一页如果内容偏少，应通过结构组件、关键数字、图表或留白平衡补足版面，而不是把内容堆在顶部。
- 图表优先服务观点，不能只做装饰。
- 同一份文件中避免重复使用同一种页面骨架；应根据内容任务选择不同版式。
- 不要在页面上放置“Excellent Operations Style”或类似设计系统署名、模板署名、风格署名。页脚只保留来源、日期、页码等对读者有用的信息。
- 每页如有来源，统一放在左下角固定位置，格式为 `来源：xxx`；所有页面的来源文本 X/Y 坐标、字号、颜色必须一致，不要放进卡片、图表内部或右下角。
- 所有正文、标签、数字和注释必须落在预设文本框内。生成前先按文本框宽度主动断句换行，生成后检查渲染结果；发现溢出、裁切、压线或越出边界时，必须改文本、字号、框高或版式。
- 文字与文字、文字与形状、文字与线条之间必须保留稳定间距。不得出现文字叠在边框、箭头、图表、分割线或其他文字上的情况。

## 质量检查

交付前必须检查：

1. HTML 与 PPTX 页数一致、页面顺序一致、核心内容一致。
2. 标题是否形成清晰的逻辑链。
3. 页数是否经过压缩，超过 12 页是否有明确理由。
4. 每页是否有明确观点，且不超过三点。
5. 字体、字号、颜色和强调方式是否符合规则。
6. 主体区域是否垂直平衡，是否出现内容贴顶、贴底或空白失衡。
7. 每页来源是否全部在左下角固定位置，且格式、字号、颜色、对齐方式一致。
8. 是否存在文字溢出、错位、遮挡、低对比度、空白失衡。
9. 是否存在占位文字、重复模板页、无意义装饰、空图表、模板署名或“Excellent Operations Style”文字。
10. 是否误用了动画或过度夸张的视觉效果。
11. 是否导出过预览图、PDF 或浏览器截图并逐页检查文字换行、遮挡和主体居中。

## 不可交付条件

出现以下情况时，必须继续修改，不能交付：

- 只有格式套用，没有根据内容选择版式。
- 页数失控，长报告被机械拆成过多页面。
- 页面重复感强，像同一页换文字。
- 主体内容没有在画布中形成垂直平衡，页面明显贴顶或空。
- 标题只是名词，不表达观点。
- 单页信息过载，用户无法快速读出重点。
- 关键内容没有视觉强调，或强调过度导致失焦。
- HTML 和 PPTX 内容不一致。
- PPTX 不可编辑，或主要内容被做成整页截图。
- 页面出现“Excellent Operations Style”等样式署名。
- 来源未放在左下角固定位置，或多页来源位置不一致。
- 任一页存在文字溢出、裁切、遮挡、压线、压框、与形状重叠。
- 主体内容整体偏上、偏下或偏侧，没有在可用画布中形成视觉居中。
- 页面内容明显过少，只靠留白、标签或装饰撑版面。
