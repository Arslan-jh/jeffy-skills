# 卓越运营 PPT 生成

使用 `assets/template.pptx` 作为唯一视觉模板，为用户主题生成规范企业风格 PPT。默认同时输出 `.pptx` 和 `.html`：PPTX 用于正式汇报和二次编辑，HTML 用于快速预览、校对标题链和页面内容。

## 默认输出

- 默认生成 12 页：1 页开头页、10 页核心内容页、1 页结束页。
- 用户指定页数时，以用户指定为准。
- 除非用户明确只要一种格式，否则同时生成 `.pptx` 和 `.html`。
- PPTX 和 HTML 必须来自同一份 deck JSON，避免内容不一致。

## 不可违反规则

- 观点在标题上，标题使用简短语句表达。
- 每页传递观点三点以内。
- 中文字体使用微软雅黑，英文字体使用 Times New Roman。
- 每页字号三种以内。
- 决不允许出现动画和页面转场。
- 页面必须符合模板视觉规范。
- 关键内容标红加粗，但只强调关键词、核心数字、风险点或行动重点。
- 慎用感叹号，默认不使用感叹号。

## 工作流

1. 理解用户主题、用途、受众、资料来源、页数和输出语言。
2. 先生成标题链，再生成每页正文。标题链必须能串成一条完整故事线。
3. 根据 `references/template-map.md` 为每页选择模板页和页面角色。
4. 将页面规划保存为统一 deck JSON。
5. 使用 `scripts/build_ppt.py` 从 `assets/template.pptx` 生成 `.pptx`。
6. 使用 `scripts/build_html.py` 从同一份 deck JSON 生成 `.html`。
7. 使用 `scripts/export_preview.py` 导出 PPT 预览图，检查错位、溢出、空白和信息过载。
8. 返回 `.pptx` 和 `.html` 文件路径，说明两者用途。

## Deck JSON

脚本使用同一份 JSON 作为输入：

```json
{
  "title": "AI客服运营效率提升方案",
  "subtitle": "管理层汇报",
  "slides": [
    {
      "template_slide": 13,
      "role": "背景判断",
      "title": "客服运营效率提升进入流程重构阶段",
      "points": [
        {
          "heading": "当前瓶颈",
          "body": ["人工处理占比高", "跨系统流转耗时长"]
        },
        {
          "heading": "优化方向",
          "body": ["智能分流前置", "标准处置流程沉淀"]
        }
      ]
    }
  ]
}
```

## 质量检查

- 每页标题是否表达明确观点，而不是栏目名。
- 每页核心观点是否不超过三点。
- 正文是否直接支撑标题。
- 是否保留模板原有字体、颜色、卡片、线条和布局。
- 是否有文字溢出、元素遮挡、页面空白或明显错位。
- 是否同时生成 PPTX 和 HTML。
