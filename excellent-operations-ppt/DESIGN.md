# 卓越运营 PPT 设计系统

## 1. 定位

卓越运营 PPT 是一套高信息密度的企业汇报设计系统，适用于经营分析、管理层汇报、战略汇报、方案汇报、运营复盘、培训材料、方法论介绍和转型项目汇报。

这套设计系统的目标不是做营销展示页，而是服务管理判断、业务沟通和执行推进。页面应当显得克制、严谨、结构清楚、信息扎实，适合反复阅读、讨论和二次编辑。

本设计系统以 `DESIGN.md` 作为规范来源，不捆绑 PPTX 文件。使用时应根据内容表达任务选择版式、组件、色彩、图表和排版方式；如果用户另行提供 PPTX，可将其作为补充素材参考，但不要机械填充。

## 2. 核心原则

- 观点在标题上。每页标题必须表达判断、结论、问题、变化、动作或建议。
- 标题用简短语句表达，避免“现状分析”“问题梳理”“解决方案”这类无观点标题。
- 每页传递观点三点以内，超过三点必须合并、分组或拆页。
- 内容先定框架，再填内容，再精简和突出重点。
- 标题链要能串成完整故事线，读者只看标题也能理解汇报主线。
- 页面要有结构表达，不要只放松散 bullet。
- 信息密度要接近模板，不能空洞，也不能把长段文字硬塞进页面。
- 优先使用模板已有版式和组件，不自行创造新的视觉语言。
- 避免连续使用同一种低密度版式。
- 不允许动画和页面转场。

## 3. 页数与内容规模

不要默认固定 12 页，也不要因为材料丰富就机械扩页。页数由内容复杂度、材料丰富度、受众和用途决定，但必须先做压缩判断，再决定是否扩展。

- 简短汇报：6-8 页。
- 标准业务汇报：8-12 页。
- 深度分析、方法论、培训、复杂方案或素材丰富型内容：13-16 页。
- 只有在用户明确要求完整展开，或内容确实包含多个不可合并的独立模块时，才扩展到 17-24 页。
- 长报告转 PPT 时，优先合并背景、证据、附录和相近机制页，不要把原文目录逐章搬进 PPT。
- 超过 12 页时，每一页都必须有不可合并的独立表达任务；如果不能说明，必须合并或删除。
- 内容不足时不要凑页数。
- 内容丰富时不要为了少页数牺牲可读性。
- 每页必须承担明确表达任务，没有表达任务的页面应删除。

推荐页数判断流程：

1. 先提炼 5-8 个必须传递的核心判断。
2. 将背景、证据、机制、风险、结论合并成若干表达任务。
3. 优先生成 8-12 页版本。
4. 只有当单页超过三点、关键图表无法容纳或逻辑链断裂时，才拆页。
5. 拆页后仍要检查是否能用矩阵、架构、时间轴或对比页重新合并。

## 4. 页面蓝图

生成 PPT 前必须先做页面蓝图。每页蓝图至少包括：

- 页面角色
- 观点标题
- 表达类型
- 主要素材来源
- 辅助组件来源
- 信息密度
- 页面结构说明
- 正文内容块
- 潜在质量风险

常见表达类型：

- 两类对比
- 三类分类
- 四类并列
- 流程路径
- 时间演进
- 能力架构
- 系统关系
- 证据矩阵
- 方法总览
- 总结收束

页面蓝图示例：

```json
{
  "page": 1,
  "role": "方法定位",
  "title": "立体分析法把复杂对象研究转化为三轴判断系统",
  "expression_type": "方法总览",
  "primary_asset": "第23页层型架构图",
  "supporting_assets": ["第5页红色编号圆点", "第5页灰框卡片"],
  "density": "高",
  "structure": "左侧解释方法定位，右侧用 X/Y/Z 分层结构呈现研究框架",
  "quality_risk": "避免只做成三张简单卡片"
}
```

## 5. 字体与字号

- 中文使用微软雅黑。
- 不使用其他文字特效。
- 英文、英文缩写和独立英文单词优先使用 Times New Roman。
- 每页字号不超过三种。
- 页面主标题使用 20-24 号字。
- 卡片一级标题使用 12 号字。
- 卡片正文使用 10 号字。
- 卡片二级标题使用 10 号字。
- 二级正文使用 9 号字。
- 标签、注释、图例、弱化信息常用 8 号字。
- 非重点辅助注释最小可使用 6 号字。
- 内容较多时，大段文字不建议使用 11 号或更大字号。

## 6. 段落样式

- 默认段落行距为 1.5 倍。
- 特殊情况下可调整为其他倍数。
- 正文段落使用两端对齐。
- 段前间距一般为 0。
- 段后间距一般为 12，可根据内容量调整为 3、6、12、18。
- 项目符号和编号一般使用默认样式。
- 避免大段文字堆叠，优先拆成短句、分组、卡片或表格。

## 6.1 页面垂直平衡

页面主体必须在标题区下方的可用画布中形成垂直平衡。默认做法是把主体内容作为一个整体，在主体区域内视觉居中，而不是从标题线下方直接堆到页面上沿。

- 标题区和页脚区固定后，主体区域应有稳定的上、下留白。
- 主体内容较少时，优先使用关键数字、结构图、对比卡、摘要条或辅助说明形成完整版面。
- 主体内容较多时，先压缩语句、合并层级或拆页，不要让内容贴近页脚。
- 横向多栏、矩阵、流程图、时间轴等版式，应保证整体模块组在画布中垂直居中。
- 单页中只有 1-2 个短 bullet 时，不得放在页面上半部了事，必须改成结构化组件或与相邻内容合并。
- 页面上下空白差异过大、主体明显贴顶、主体贴底、下半屏长期空白，均视为版式失败。

## 7. 色彩系统

### 主红

- RGB 199 / 0 / 11
- RGB 192 / 0 / 0

主红用于页面标题、重点强调、红色标题栏、主箭头和核心视觉元素。

### 红色系

- RGB 213 / 60 / 68
- RGB 227 / 120 / 125
- RGB 241 / 180 / 182
- RGB 255 / 241 / 239

红色系用于图表渐变、浅色强调底、提示背景和辅助强调。

### 灰色系

- RGB 140 / 140 / 140
- RGB 166 / 166 / 166
- RGB 191 / 191 / 191
- RGB 217 / 217 / 217
- RGB 242 / 242 / 242

灰色用于边框、分割线、连接线、弱化信息、非激活状态、辅助图形和背景模块。

### 辅助色

蓝色、黄色、绿色只在需要区分类别时使用，例如架构图、多分类图表、对象分组。

蓝色系：

- RGB 17 / 92 / 170
- RGB 72 / 131 / 191
- RGB 127 / 170 / 212
- RGB 182 / 209 / 233
- RGB 238 / 248 / 254

黄色系：

- RGB 252 / 200 / 0
- RGB 255 / 212 / 49
- RGB 255 / 223 / 100
- RGB 255 / 234 / 152
- RGB 255 / 244 / 203

绿色系：

- RGB 97 / 178 / 48
- RGB 133 / 196 / 96
- RGB 170 / 214 / 144
- RGB 207 / 232 / 192
- RGB 244 / 250 / 240

## 8. 文本颜色与强调

- 常规正文使用黑色。
- 强调内容使用红色。
- 非重点内容、注释、辅助标签使用灰色。
- 红色标题栏中的标题使用白色反白。
- 页面标题、卡片一级标题可以加粗。
- 关键数字、关键词、风险点、行动重点可以标红加粗。
- 小标题和正文不建议频繁加粗。
- 禁止整段正文加粗或整段正文标红。
- 如果一页中强调内容过多，必须减少到 1-3 个最重要的信息点。

## 9. 卡片与组件

### 卡片

- 卡片轮廓以浅灰框、深灰框、红色框为主。
- 卡片填充以白色、浅灰、红色系为主。
- 红头卡片用于核心模块、一级分组和强强调信息。
- 灰框卡片用于正文内容、支撑信息和次级模块。
- 卡片内文本需要保留稳定边距。
- 不使用项目符号或编号时，左右边距保持一致。
- 常用左右边距为 0.4、0.6、0.8 厘米。
- 使用项目符号或编号时，左边距通常比右边距少 0.2 厘米。
- 禁止文本贴近卡片边框。

### 线条与箭头

- 箭头线、连接线、分割线、卡片轮廓线优先使用 0.5 磅。
- 线条颜色优先使用浅灰、深灰、黑色或红色。
- 红色箭头只用于主方向、关键推进或核心流程。
- 灰色线条用于弱连接、辅助关系、分组边界和跨模块关联。
- 箭头大小一般选择最小。
- 不要在同一页混用过多线条粗细。

### 标签与 Tab

- 红色用于激活态、重点标签或当前分类。
- 灰色用于非激活态、辅助标签或次级分类。
- 标签必须帮助理解分组、状态、阶段或类别。
- 不要把 tab 当作无意义装饰。

### 编号与指标

- 红色圆点或方块编号适合步骤、优先级和流程节点。
- 红色数字适合核心指标，灰色文字用于指标解释。
- 指标必须服务页面结论，不要为了填充页面而造指标。

## 10. 图表规则

- 柱状图优先使用红色系或灰色系。
- 对比类图表使用红色突出关键系列，其余系列用灰色弱化。
- 饼图、环图优先使用同一色系的深浅变化。
- 图表标题栏可使用红底白字。
- 图表标签、图例字号应低于正文层级。
- 图表颜色必须服务于结论，不为了装饰而多色。
- 除非需要表达明确分类，否则不使用多色图表。

## 11. 版式类型

### 两类对比

适合现状/问题、问题/建议、方案/收益、两类对象并列说明。

- 左右两栏必须同层级。
- 每栏 1 个标题和 2-3 条支撑信息。
- 可补充标签、指标卡或红框卡片。
- 不适合三类以上并列或时间演进。

### 三类分类

适合三类问题、三项举措、三阶段路径、三类能力。

- 三栏必须同层级。
- 每栏 1 个标题和 2-3 条支撑信息。
- 避免某一栏内容明显过重。

### 四类并列

适合四类指标、四个模块、四项能力、四个对象。

- 每个模块保持短信息。
- 每区 1 个标题和 1-2 条正文或 1 个关键指标。
- 不适合长解释。

### 流程路径

适合端到端流程、执行步骤、业务链路和协同机制。

- 流程必须有明确方向。
- 节点用短词。
- 长解释放到旁侧说明区。
- 不要把无方向的分类内容做成流程。

### 时间演进

适合年度规划、阶段路径、里程碑和演进过程。

- 通常使用 3-4 个阶段。
- 每阶段包含短标题、关键动作和阶段结果。
- 不适合非时间顺序内容。

### 能力架构

适合能力分层、业务架构、平台架构、运营体系和方法框架。

- 层级必须自上而下或自下而上有清晰逻辑。
- 每层代表不同抽象层级。
- 不要把并列分类伪装成架构层级。

### 系统关系

适合系统关系、模块关系、平台能力图和复杂对象连接。

- 只在确实需要表达复杂关系时使用。
- 模块标签要短。
- 连接线要克制，避免视觉噪音。

### 证据矩阵

适合证据分层、来源比较、质量检查、决策标准。

- 表头可使用红底白字。
- 表格线使用浅灰。
- 单元格内容必须短。
- 不要把表格做成长文档。

## 12. 素材选择

使用本设计系统时，先判断表达任务，再选择素材和版式：

| 表达任务 | 优先素材 | 可补充组件 |
|---|---|---|
| 两类对比 | 二分栏页面 | 标签、指标卡、红框卡片 |
| 三类分类 | 三分栏页面 | 红色编号、灰框卡片 |
| 四类并列 | 四分栏或四卡片页面 | 图标、指标卡 |
| 流程路径 | 流程图页面 | 箭头、编号圆点、流程卡片 |
| 时间演进 | 时间轴页面 | 年份标签、红色箭头 |
| 能力架构 | 层型架构页面 | 层级卡片、红色核心模块 |
| 系统关系 | 逻辑架构页面 | 模块框、连接线、应用标签 |
| 证据矩阵 | 四分栏或表格组件 | 红头表格、灰色单元格 |
| 方法总览 | 架构页面或三分栏页面 | 核心节点、分层模块 |
| 总结收束 | 四卡片或二分栏页面 | 红头卡片、行动标签 |

## 13. 质量标准

交付前必须检查：

- 标题是否表达明确观点，而不是栏目名。
- 标题链是否形成完整故事线。
- 页数是否由内容复杂度决定，而不是机械固定。
- 页数是否先经过压缩判断，超过 12 页是否有独立表达任务支撑。
- 每页是否有明确表达任务。
- 是否避免连续重复同类版式。
- 是否充分调用模板中的整页版式和组件素材。
- 每页核心观点是否不超过三点。
- 正文是否直接支撑标题。
- 主体内容是否在页面可用画布中垂直平衡。
- 是否保留模板原有字体、颜色、卡片、线条和布局语言。
- 是否有文字溢出、元素遮挡、页面空白或明显错位。
- 是否清理了原模板占位文字。

## 14. 不可交付条件

出现以下任一情况时，不得交付，必须重做或修正：

- 页面明显空洞，只放 1-2 个短 bullet 且没有结构表达。
- 长报告被机械拆成过多页面，超过 12 页但没有逐页说明独立表达任务。
- 多页连续使用同一种低密度版式。
- 整份 PPT 没有使用流程图、时间轴、架构图、矩阵或多栏结构中的至少两类。
- 没有使用模板中的页面版式或组件素材。
- 原模板占位文字未清理干净。
- 文字压住线条、图表、卡片或其他图形。
- 主体内容明显贴近页面上沿、下沿或没有视觉居中。
- 标题不是观点句。
- 页面内容不能支撑标题。
- 导出的预览图存在文字溢出、遮挡、裁切或明显错位。
- 页面风格变成普通白卡片，失去模板的信息密度和组件语言。

## 15. 禁止事项

- 不要制作通用营销落地页式 hero。
- 不要使用大面积装饰性渐变。
- 不要让页面只有大标题和少量 bullet。
- 不要大量使用红色。
- 不要滥用蓝色、黄色、绿色。
- 不要使用感叹号，除非是强提醒或风险警示。
- 不要为了显得复杂而使用架构图。
- 不要为了填充页面而编造指标。
- 不要交付错位、遮挡、溢出或占位符残留的页面。
